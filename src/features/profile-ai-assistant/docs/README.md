# AI-помощник создания профиля

Фича содержит интерфейс правой AI-панели в сценарии создания профиля должности.

Продуктовая спецификация находится в:

```text
docs/components/profile-ai-assistant/product-spec.md
```

## Роль фичи

AI-помощник должен сопровождать пользователя на двух этапах создания профиля:

- `Общие положения и функционал`;
- `Ключевые компетенции`.

Панель открывается по кнопке `ИИ-Помощник` в хедере мастера создания профиля и не должна сбрасываться при переключении этапов.

## Целевая структура панели

Верхняя навигация панели должна быть реализована как segmented control:

```text
Генерация | Анализ | Чат
```

`Генерация`
Создает черновик профиля по свободному тексту или предиктивно заполняет компетенции на основе функций первого этапа.
Визуально должна восприниматься как action-инструмент: `Сгенерировать`, loader создания, результат с будущими изменениями и смысловой бейдж `Может изменить профиль после подтверждения`.

`Анализ`
Проверяет заполненный профиль, показывает ошибки, предупреждения и рекомендации, синхронизирует карточки анализа с полями основной формы.

`Чат`
Отвечает на вопросы по методологии профилирования и показывает контекстные подсказки по текущему полю.
Визуально должен восприниматься как безопасная консультация: `Отправить`, вопросные чипсы, текстовые ответы и смысловой бейдж `Только консультация`.

## Ожидаемая ответственность файлов

`model/profile-ai-assistant.analysis-card.model.js`
Хранит техническую модель карточки анализа: допустимые типы, группы критичности, этапы, виды целей навигации, типы действий, фабрику карточки, базовую проверку и правило разделения карточек, если в них одновременно есть предложение и действие.

`data/profile-ai-assistant.analysis-card.mock.js`
Хранит JSON-подобные примеры карточек анализа для документации, проверки структуры и будущего прототипирования. Эти данные не управляют текущей логикой анализа, а служат витриной возможных карточек.

`data/profile-ai-assistant.analysis-card.examples.json`
Хранит человеко-читаемый каталог примеров карточек анализа. Сейчас файл не подключается к интерфейсу и не является вторым источником истины; он нужен для документации, обсуждения атрибутов и будущей синхронизации с мок-данными.

`data/profile-ai-assistant.mock.js`
Хранит демонстрационные данные: быстрые промпты, hardcoded-ответы чата, сценарии генерации, правила анализа и демо-рекомендации.

`state/profile-ai-assistant.state.js`
Хранит состояние панели: открытость, ширину, активную вкладку, историю чата, черновик ввода, состояние лоудеров, карточки анализа и примененные AI-исправления.

`render/profile-ai-assistant.render.js`
Отвечает за отрисовку панели, segmented control, вкладок, сообщений, quick prompts, карточек анализа, empty states и loader-состояний.

`events/profile-ai-assistant.events.js`
Подключает события: открытие панели, переключение вкладок, отправку промпта, применение рекомендаций, undo, отслеживание фокуса в форме и реакцию на смену этапа создания профиля.

`styles/profile-ai-assistant.css`
Содержит стили панели, segmented control, сообщений, чипсов, карточек анализа, AI-меток, подсветок и loader-состояний.

## Важные правила

- AI не меняет данные формы без явного действия пользователя.
- Индикация анализа в рабочей области видна только при активной вкладке `Анализ`.
- Клик по рекомендации должен сохранять предыдущее значение для undo.
- Если пользователь исправил поле вручную, карточка анализа должна уметь перейти в состояние `Выполнено`.
- На втором этапе генерация должна опираться на задачи и функции первого этапа.
