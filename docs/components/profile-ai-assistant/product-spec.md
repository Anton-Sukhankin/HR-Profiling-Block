# AI-помощник создания профиля: продуктовая спецификация

AI-помощник — правая боковая панель внутри интерфейса создания профиля должности. После отказа от отдельного режима создания структуры профиля панель содержит два режима:

```text
Анализ | Чат
```

## Цель

AI-помощник должен помогать пользователю качественно заполнить профиль должности без перегрузки интерфейса:

- выявлять критические ошибки, предупреждения и рекомендации;
- связывать замечания с конкретными элементами рабочей области;
- позволять выполнить безопасные действия прямо из анализа;
- давать возможность отката выполненных действий;
- отвечать на вопросы по методологии профилирования.

## Границы

В интерфейсе AI-помощника больше нет отдельной вкладки для создания или массового предзаполнения профиля по текстовому запросу.

Текущие границы:

- `Анализ` может навигировать по рабочей области, применять конкретные действия и возвращать прежние значения.
- `Чат` отвечает на вопросы и не меняет данные формы.
- Второй этап не открывается и не заполняется через AI, пока первый этап не дал минимальный контекст.

## Вкладка «Анализ»

Вкладка `Анализ` работает как Live Validation и методологический аудитор.

Анализ проверяет:

- обязательный контекст первого этапа;
- наличие цели, задачи и функции;
- качество декомпозиции задач;
- корректность и полноту атрибутов второго этапа, если он доступен;
- соответствие компетенций функционалу первого этапа.

### Минимальный контекст

Минимальный контекст для разблокировки второго этапа и анализа компетенций:

1. выбрана должность;
2. выбрано место в структуре;
3. есть непустая цель;
4. есть непустая задача;
5. выбрана хотя бы одна функция.

Если контекста не хватает, анализ показывает empty state или карточки, объясняющие, какие данные нужно заполнить вручную.

### Группы анализа

Карточки анализа группируются по критичности:

- `Критические ошибки` — блокирующие или контекстно-критичные проблемы;
- `Предупреждения` — проблемы качества заполнения;
- `Рекомендации` — предложения по обогащению и уточнению профиля;
- `Исправлено` — отдельная структурная группа для карточек с типом `success`, если такие карточки явно созданы моделью.

Выполненные карточки типов `critical`, `warning` и `recommendation` остаются в своих исходных группах, меняют состояние на `done` и показывают действие возврата. Они не переносятся автоматически в группу `Исправлено`.

Группы отображаются как аккордеоны. Если в группе есть карточки с действиями, в хедере группы доступна кнопка массового выполнения действий. Если все действия группы выполнены, кнопка меняется на возврат.

### Карточка анализа

Карточка анализа содержит:

- тип и визуальную индикацию;
- тайтл проблемы;
- описание;
- опциональное предложение;
- опциональное действие;
- состояние выполненного действия;
- undo, если действие обратимо.

Одна активная карточка должна содержать либо предложение, либо действие. Если для одного замечания нужны и предложение, и действие, это лучше разделять на две карточки.

### Действия

Действие выполняется только по явному клику пользователя.

Поддерживаемые типы:

- переход к связанному полю или элементу;
- применение конкретного значения;
- добавление значения в аккордеон второго этапа;
- удаление значения;
- замена одного значения другим;
- массовое выполнение действий внутри группы.

После выполнения действия карточка не исчезает. Она остается видимой, показывает done-состояние и блок возврата предыдущего значения.

### Индикации рабочей области

Пока активна вкладка `Анализ`, связанные поля рабочей области подсвечиваются бордером соответствующего цвета:

- критическая ошибка — красный;
- предупреждение — янтарный;
- рекомендация — синий или фиолетовый.

При уходе во вкладку `Чат` индикации анализа скрываются, чтобы не перегружать рабочую область.

## Вкладка «Чат»

Вкладка `Чат` — консультационный режим.

Чат может:

- отвечать на вопросы о методологии целей, задач, функций и компетенций;
- объяснять шкалы и уровни;
- пояснять, почему анализ показывает конкретное замечание;
- помогать пользователю принять решение;
- показывать быстрые вопросы под приветственным сообщением.

Чат не должен:

- менять данные формы;
- применять значения;
- открывать второй этап;
- выполнять действия карточек анализа.

## Память панели

Панель должна оставаться открытой при переключении между этапами создания профиля.

При выходе из интерфейса создания профиля:

- временные результаты анализа очищаются;
- переписка чата может сохраняться в рамках прототипа;
- удалённый режим создания структуры профиля не восстанавливается даже при наличии старого состояния в `localStorage`.

## Имитация AI в прототипе

Реальная LLM в прототипе не подключена. Поведение имитируется через:

- анализ DOM-состояния формы;
- моковые ответы чата;
- заранее заданные карточки и действия;
- короткие задержки для loader-состояний.

Несмотря на имитацию, пользовательский опыт должен выглядеть как работа помощника, который понимает контекст профиля и методологию заполнения.

## Открытые вопросы

- Нужно ли в будущем возвращать создание черновика профиля как отдельный сценарий вне текущей AI-панели?
- Должен ли анализ предлагать более сложные составные действия или только простые точечные исправления?
- Где должна храниться история консультаций чата при работе с несколькими профилями?
