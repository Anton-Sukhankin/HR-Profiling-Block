# Модель карточки анализа AI-помощника

Документ фиксирует атрибутивный состав сущности `AnalysisCard` для вкладки `Анализ` в интерфейсе AI-помощника создания профиля должности.

Эта спецификация не меняет текущую функциональность и верстку. Она нужна как контракт для дальнейшей разработки: чтобы карточки анализа, навигация по рабочей области, предложения AI и действия развивались по единой модели.

## Назначение

`AnalysisCard` описывает одну индикацию, найденную AI-помощником при анализе профиля.

Карточка может относиться:

- к первому этапу `Общие положения и функционал`;
- ко второму этапу `Ключевые компетенции`;
- к конкретному полю, аккордеону, строке задачи, функции или карточке сущности;
- к методологической проблеме, предупреждению или рекомендации.

## Базовая структура

```js
AnalysisCard = {
  id: string,
  type: "critical" | "warning" | "recommendation" | "success",
  title: string,
  description: string,
  location: string,
  targetKey: string,
  stage: "functional" | "competencies",
  targetKind: string,
  suggestedText?: string,
  suggestedValue?: string,
  action?: AnalysisCardAction,
  status?: "active" | "done",
  originalValue?: string,
  appliedValue?: string,
  undoAction?: object,
  undoLabel?: string
}
```

## Атрибуты

`id`
Уникальный технический идентификатор карточки. Используется для обновления состояния, применения предложения, undo и подсветки конкретной карточки.

`type`
Тип индикации. Определяет визуальную метафору и приоритет.

Допустимые значения:

- `critical` — критическая ошибка или блокирующее нарушение;
- `warning` — предупреждение, нехватка данных или методологический риск;
- `recommendation` — рекомендация AI или оптимизация;
- `success` — исправлено или соответствует правилу.

`title`
Короткое наименование проблемы или рекомендации. Показывается в хедере карточки.

`description`
Пояснение проблемы человеческим языком. Должно объяснять, почему карточка появилась и что пользователь должен понять.

`location`
Логическая локация внутри профиля. Может храниться в данных, но не обязана выводиться в интерфейсе, потому что клик по карточке выполняет навигацию к связанному элементу.

`targetKey`
Техническая связь карточки с элементом рабочей области. По этому ключу работают:

- AI-маркер в рабочей области;
- подсветка поля или аккордеона;
- скролл к элементу;
- переход между этапами перед скроллом.

`stage`
Этап интерфейса создания профиля, к которому относится карточка.

Допустимые значения:

- `functional` — первый этап;
- `competencies` — второй этап.

Если карточка относится ко второму этапу, клик по ней должен сначала переключить рабочую область на `Ключевые компетенции`, а затем выполнить скролл к связанному элементу.

`targetKind`
Тип целевого элемента. Нужен для будущей более точной навигации и аналитики.

Примеры:

- `position`;
- `structure`;
- `goal`;
- `task`;
- `function`;
- `softSkills`;
- `hardSkills`;
- `language`;
- `technology`;
- `education`;
- `experience`;
- `functionalArea`;
- `accordion`.

## Нижний блок карточки

Карточка анализа должна содержать только один смысловой нижний блок:

- либо `suggestedText`;
- либо `suggestedValue`;
- либо `action`.

Одна карточка не должна одновременно показывать предложение и действие.

Если одна логическая проблема требует и предложения, и действия, она должна быть разделена на две карточки:

- карточка-предложение;
- карточка-действие.

## Группировка карточек

Карточки во вкладке `Анализ` группируются по степени критичности методологических отклонений.

Группа не является большой карточкой-контейнером. В интерфейсе это лаконичный блок аккордеона, который визуально разделяет поток карточек:

- лейбл группы;
- счетчик карточек;
- короткое пояснение;
- верхний и нижний бордер;
- верхний и нижний паддинг;
- управление раскрытием группы как аккордеона.

Иконка у группы не выводится, чтобы не конкурировать с иконками внутри карточек.

Текущие группы:

```js
AnalysisGroup = {
  id: string,
  type: "critical" | "warning" | "recommendation" | "success",
  title: string,
  description: string,
  priority: number,
  blocksNextStep: boolean
}
```

`critical`
Критические ошибки. Блокируют переход дальше.

`warning`
Предупреждения. Влияют на качество заполнения, но не блокируют переход.

`recommendation`
Рекомендации. Помогают обогатить профиль.

Если внутри группы есть карточки с действиями, группа может показывать компактную кнопку массового действия.

Кнопка массового действия применяет все доступные действия карточек внутри этой группы одним нажатием. После массового применения кнопка может переключаться в состояние отката, если примененные карточки поддерживают undo.

Справа от заголовка группы также находится кнопка раскрытия/сворачивания аккордеона. В раскрытом состоянии карточки видны, в свернутом состоянии карточки скрыты.

## Предложение

`suggestedText`
Текстовое предложение AI. Оно не является кнопкой и не применяет изменения к форме.

Визуальные правила:

- светло-серая заливка;
- без бордера;
- без hover-эффекта;
- обычный курсор;
- иконка-метафора предложения, например лампочка.

`suggestedValue`
Конкретное значение, которое можно применить к связанному полю.

В отличие от `suggestedText`, это интерактивный вариант значения. При клике значение может быть подставлено в целевое поле, если такая механика предусмотрена карточкой.

## Действие

`action`
Описывает действие, которое пользователь может явно запустить.

```js
AnalysisCardAction = {
  type: "focus_target" | "switch_generation" | "apply_suggestion" | "apply_competency_value" | "custom",
  label: string,
  prompt?: string,
  section?: string,
  mode?: "add" | "remove" | "replace" | "set_level",
  value?: string,
  addValue?: string,
  removeValue?: string,
  language?: string,
  level?: string
}
```

`type`
Тип действия.

Допустимые значения:

- `focus_target` — перейти к связанному элементу рабочей области;
- `switch_generation` — перейти во вкладку `Генерация` и подготовить запрос;
- `apply_suggestion` — применить предложенное значение;
- `apply_competency_value` — добавить, удалить, заменить или установить значение на этапе `Ключевые компетенции`;
- `custom` — будущий расширяемый сценарий.

`label`
Текст действия в карточке.

`prompt`
Опциональный текст, который может быть подставлен во вкладку `Генерация`.

Визуальные правила для действия:

- белая заливка;
- серый бордер;
- синий бордер при наведении;
- синяя иконка на голубой подложке;
- курсор и hover должны показывать, что блок кликабельный.

## Состояния

`active`
Карточка актуальна и требует внимания пользователя.

`done`
Карточка считается выполненной. Это возможно, если:

- пользователь применил `suggestedValue`;
- пользователь сам ввёл рекомендуемое значение в рабочей области;
- проблема больше не воспроизводится при повторном анализе.

`originalValue`
Значение поля до применения предложения. Нужно для undo.

`appliedValue`
Значение, которое было применено.

`undoAction`
Структурированное действие возврата предыдущего состояния. Используется для отката массовых действий, генерации задач/функций и изменений компетенций.

`undoLabel`
Текст блока возврата, например `Вернуть предыдущее значение`.

## Навигация

Клик по карточке должен вести пользователя к связанному элементу рабочей области.

Если карточка относится к текущему этапу:

- выполнить скролл к элементу;
- подсветить элемент;
- оставить AI-панель открытой.

Если карточка относится к другому этапу:

- переключить рабочую область на нужный этап;
- синхронизировать AI-маркеры;
- выполнить скролл к элементу;
- подсветить элемент.

Это правило одинаково для карточек первого и второго этапов.

## Синхронизация с рабочей областью

Пока активна вкладка `Анализ`, целевой элемент в рабочей области получает AI-индикацию.

Индикация должна соответствовать `type` карточки:

- `critical` — красный акцент;
- `warning` — оранжевый акцент;
- `recommendation` — синий или фиолетовый акцент;
- `success` — зелёный акцент.

Если несколько карточек связаны с одним элементом, в рабочей области должен отображаться один маркер, чтобы не создавать визуальный шум.

## Правила полноты карточки

Карточка считается полной, если содержит:

- `id`;
- `type`;
- `title`;
- `description`;
- `targetKey`;
- один нижний смысловой блок: `suggestedText`, `suggestedValue` или `action`.

Карточка, состоящая только из `title` и `description`, нежелательна для UX. Исключение возможно только для информационного состояния, которое не требует реакции пользователя.

## Примеры

Карточка-предложение:

```js
{
  id: "analysis_soft_skills_relevance",
  type: "recommendation",
  title: "Проверьте связь Soft Skills с функциями",
  description: "Soft Skills должны объяснять способ выполнения работы, а не быть универсальным набором качеств.",
  location: "Ключевые компетенции → Компетенции",
  targetKey: "analysis_soft_skills_relevance",
  stage: "competencies",
  targetKind: "softSkills",
  suggestedText: "Оставьте только те soft skills, которые реально помогают выполнять выбранные функции первого этапа."
}
```

Карточка-действие:

```js
{
  id: "analysis_hard_skills_match",
  type: "recommendation",
  title: "Проверьте соответствие Hard Skills функциям",
  description: "Профессиональные навыки должны подтверждаться задачами и функциями профиля.",
  location: "Ключевые компетенции → Профессиональные знания и навыки",
  targetKey: "analysis_hard_skills_match",
  stage: "competencies",
  targetKind: "hardSkills",
  action: {
    type: "focus_target",
    label: "Перейти к Hard Skills"
  }
}
```

Карточка со значением:

```js
{
  id: "analysis_goal_0_missing_name",
  type: "warning",
  title: "Цель не выбрана",
  description: "Цель задаёт верхнеуровневый смысл профиля и нужна для дальнейшего анализа.",
  location: "Цель 1",
  targetKey: "analysis_goal_0_missing_name",
  stage: "functional",
  targetKind: "goal",
  suggestedValue: "Обеспечение анализа и развития бизнес-процессов",
  originalValue: ""
}
```

## Текущая реализация в прототипе

На текущем этапе прототип уже использует большинство полей этой модели:

- `id`;
- `type`;
- `title`;
- `description`;
- `location`;
- `targetKey`;
- `suggestedText`;
- `suggestedValue`;
- `action`;
- `status`;
- `originalValue`;
- `appliedValue`;
- `undoAction`;
- `undoLabel`.

Поля `stage` и `targetKind` являются частью контрактной модели карточки.

`stage` фиксирует этап, к которому относится карточка:

- `functional`;
- `competencies`;
- `global`.

`targetKind` уточняет тип связанного элемента рабочей области:

- `field`;
- `select`;
- `task`;
- `function`;
- `accordion`.

Runtime-навигация пока дополнительно опирается на связанный DOM-элемент через `targetKey`, но данные карточки должны сохранять этап и тип цели явно, чтобы карточки анализа могли корректно работать между двумя этапами создания профиля.

## Файлы модели в структуре проекта

Техническое описание модели вынесено в файл:

```text
src/features/profile-ai-assistant/model/profile-ai-assistant.analysis-card.model.js
```

Примеры карточек анализа вынесены в файл:

```text
src/features/profile-ai-assistant/data/profile-ai-assistant.analysis-card.mock.js
```

Человеко-читаемый JSON-каталог примеров вынесен в файл:

```text
src/features/profile-ai-assistant/data/profile-ai-assistant.analysis-card.examples.json
```

`model.js` нужен для правил, допустимых значений и базовой проверки структуры.

`mock.js` нужен для JSON-подобных примеров конкретных карточек, которые проще читать как готовые данные.

`examples.json` нужен для документации и обсуждения структуры карточек без необходимости читать JavaScript. Он не подключается к интерфейсу и не должен рассматриваться как второй источник истины для текущего прототипа.
