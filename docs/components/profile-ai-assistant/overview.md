# AI-помощник: обзор компонента

Компонент `profile-ai-assistant` — правая AI-панель внутри интерфейса создания профиля должности. Он сопровождает пользователя на двух этапах создания профиля:

- `Общие положения и функционал`;
- `Ключевые компетенции`.

Панель открывается по кнопке `ИИ-Помощник` в хедере мастера создания профиля и должна оставаться открытой при переключении между этапами.

## Назначение

AI-помощник снижает сложность заполнения профиля, помогает сформировать черновик, проверяет методологическое качество данных и объясняет правила профилирования в контексте текущего профиля.

Компонент объединяет три режима в segmented control:

- `Генерация` — может создавать или предзаполнять значения профиля.
- `Анализ` — проверяет текущие данные, показывает критические ошибки, предупреждения и рекомендации, может применять действия и возвращать прежние значения.
- `Чат` — отвечает на вопросы по методологии и текущему профилю, но не меняет данные формы.

## Границы компонента

AI-помощник не является самостоятельной страницей. Он работает поверх интерфейса создания профиля и зависит от данных рабочей области.

Компонент не должен напрямую ломать или обходить правила создания профиля. Если AI предлагает действие, оно должно выполняться через согласованную механику интерфейса: явный клик пользователя, изменение данных, состояние выполненной карточки и возможность отката, если действие обратимо.

## Ключевые принципы

- AI не меняет данные без явного действия пользователя.
- Вкладка `Чат` не мутирует данные формы.
- Вкладка `Анализ` показывает индикации рабочей области только пока активна.
- Второй этап не должен заполняться или открываться через AI, если первый этап не дал минимальный контекст.
- Документация компонента разделяется по смысловым слоям: сценарии, бизнес-правила, UI-взаимодействия, AI-логика, модель данных и интеграции.
